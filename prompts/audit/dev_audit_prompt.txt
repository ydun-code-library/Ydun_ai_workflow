# DEVELOPMENT CODEBASE AUDIT - CLAUDE CODE
## Jimmy's Workflow: Fast Iteration + Actionable Fixes

## CONTEXT
Development audit for **shipping faster with fewer bugs**. Identifies velocity blockers, suggests fixes, prioritizes by impact, focuses on "what's slowing us NOW".

## DIRECTIVES
SPEED: Fast insights > exhaustive | ACTIONABLE: Every finding = fix | PRIORITIZE: Velocity > perfection | VERIFY: Streamlined checkpoints | ITERATE: Weekly/per-feature

## PHILOSOPHY
**Production:** Quarterly risk assessment → "All problems" → You decide
**Development:** Weekly velocity → "What's slowing you" → Fix these 3 today
**Micro-Checkpoints:** 🔴 Scan → 🟢 Verify → 🔵 Fix (5-15min each)

## PHASE 1: RECON (5min)

```bash
find . -type f -name "*.{js,ts,py}" | wc -l > /tmp/file_count.txt
find . -name "*.test.*" | wc -l > /tmp/test_count.txt
grep -r "console.log" src/ 2>/dev/null | wc -l > /tmp/console_count.txt
grep -r "TODO\|FIXME" src/ 2>/dev/null | wc -l > /tmp/todo_count.txt
echo "Files: $(cat /tmp/file_count.txt) Tests: $(cat /tmp/test_count.txt) Logs: $(cat /tmp/console_count.txt) TODOs: $(cat /tmp/todo_count.txt)"
```
**Assess:** Test ratio <0.3? Logs >20? TODOs >50? → Issues

## PHASE 2: BLOCKERS (30min)

**CRITICAL INSTRUCTION FOR LLM:**
For each BLOCKER (0-5), you MUST:
1. Execute the **🔴 SCAN** commands and save results to `/tmp/` files
2. **🟢 VERIFY** by counting results and spot-checking top 3 examples from each file
3. **🔵 GENERATE FIXES** that reference specific file:line from the `/tmp/` files with exact code changes
4. Fixes must be concrete, traceable, and based on scan evidence (not generalized suggestions)
5. Prioritize top 3 most severe examples per blocker for the final report

Example of proper fix: ❌ "Fix type errors" → ✅ "src/auth.ts:45: Change `user: any` to `user: User | null`"

### BLOCKER 1: AI Patterns (10min)

**🔴 SCAN:**
```bash
# Fingerprints: verbose names, over-commenting >30%, console.log spam, excessive try-catch, import chaos >10, mixed naming, redundant types
grep -rn "temporaryVariable\|helperFunction" src/ > /tmp/ai_verbose.txt
find src/ -name "*.js" | while read f; do CODE=$(grep -cv "^[[:space:]]*//\|^[[:space:]]*\*" "$f"); COMM=$(grep -c "^[[:space:]]*//\|^[[:space:]]*\*" "$f"); [ "$CODE" -gt 0 ] && RATIO=$((COMM*100/CODE)) && [ "$RATIO" -gt 30 ] && echo "$f: $RATIO%"; done > /tmp/ai_comment.txt
# Tangles: duplicate implementations, circular deps, unused params
find src/ -name "*.js" | xargs grep -h "function.*validate" | sort | uniq -c | awk '$1>2' > /tmp/ai_dups.txt
npx madge --circular src/ 2>/dev/null | grep -v "✓" > /tmp/ai_circular.txt
```

**🟢 VERIFY:** Count results, spot-check top 3 from each /tmp/ file

**🔵 FIXES:** Reference counts and line numbers from /tmp/ files to generate top 3 most severe, specific fixes per pattern.

**Example Fixes:**
- src/utils/parser.js:45 (47% comments, ai_comment.txt) → Remove obvious comments, simplify to: `function parse(data) { return JSON.parse(data); }`
- src/auth/validate.ts + src/users/validate.ts + src/api/validators.ts (ai_dups.txt) → Consolidate to `src/utils/validators.ts: export function validateEmail(email: string) {...}`, delete duplicates
- src/auth.ts ↔ src/user.ts (ai_circular.txt) → Break cycle: Create `src/services/auth-service.ts`, both import from service
- src/components/Dashboard.tsx:1-18 (ai_import_chaos.txt, 18 imports) → Split into Dashboard.tsx (UI, 5 imports), useDashboardData.ts (data, 3 imports), dashboardUtils.ts (helpers, 2 imports)

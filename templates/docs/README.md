# Documentation Templates

Standards and reusable components for AI-optimized project documentation.

| File | What it does |
|------|-------------|
| `DOCUMENTATION-STANDARDS.md` | The 7 principles for writing docs that both humans and AI can use |
| `AI-DRIVEN-SEO-BEST-PRACTICES.md` | SEO guide optimized for AI/LLM discovery — JSON-LD, llms.txt, entity clarity |
| `doc-components/` | Reusable templates for navigation headers, ADRs, metadata blocks, docs maps |

## doc-components/

| File | What it does |
|------|-------------|
| `ADR-TEMPLATE.md` | Architecture Decision Record — documents why decisions were made |
| `AI-NAVIGATION-HEADER.template` | Header block that tells AI assistants when to read a file |
| `DOCS-MAP.md.template` | Master navigation map for project documentation |
| `METADATA-BLOCK.template` | Machine-readable metadata for documentation files |

Best for projects with 5+ documentation files. Smaller projects can skip doc-components.

**Docs:** [docs.ydun.io/workflow/documentation-standards](https://docs.ydun.io/workflow/documentation-standards/)
